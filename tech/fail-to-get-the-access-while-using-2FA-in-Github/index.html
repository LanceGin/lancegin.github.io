Fail to get the access while using 2FA in Github

Sep 13, 2015 | tech | Hits

Nowadays, Github set a 2-factor authentication, that means if you want to login or push some codes to your repo by HTTPS, you should input 2 keys to authenticate yourself.

But the version of Git has not caught the step of Github, if you push codes to your repo in terminal by Git, it will never let you go.

The problem bordered me harder, and today I found a way to solve it. We can use an access token to ignore the 2FA.

• first you can go to the setting page to find the button to generate an access token

• then make sure the repo:status has been checked

• now you will see your new access token that will be used to ignore the 2FA.

• from then on, while you want to push or pull codes from your repo, you should tap git config --global credential.helper store first, then tap your username and your new access token.

Then the 2FA problem will not border you anymore. And, your Github will be safer.
